# Demo video script

Target: 2:35–2:55, public YouTube, clear spoken audio. Record at 1920×1080 with system notifications hidden (Codeville's own dock badge is fine to show).

**Premise: the village is my real working life.** The five lots hold five of my actual repositories with real tasks. The one-click demo village stays what it was designed as — the judges' reproduction path (see `JUDGE_TESTING.md`) — and gets one spoken mention at the end. The video itself is live proof on real products.

The five: `graphletter`, `bar-playbook`, `codeville`, `spotify-history`, `kalshi-mlb`. The real-repo flow is already proven live (2026-07-19 dogfood on two real projects), and `codeville` on its own lot is the meta-beat — the tool improving itself on camera.

## 0:00–0:18 — Hook: wall mode over my real repos

**Visual:** Cold open in **Wall mode** — five builders working on five named lots (my real repository names on the nameplates), one raises its lantern, the signpost reads "1 builder needs you".

**Voice:** "These are my actual projects — five real Codex agents working right now on the repositories I ship from. And this screen is safe to leave on a projector, because the village provably never renders code, paths, or prompts — only who's working and who needs me. When a builder needs a decision, it walks to the gate and raises a lantern."

## 0:18–0:48 — Real repositories, explicit preflight

**Visual:** Press W to the Foreman's Desk. Show `gpt-5.6-sol` and the Codex version in the header. Show one lot being assigned (**Choose…** → a real repository), skill chips equipped, the task typed. Then check several real-project cards, click **Start selected builders**, and linger one beat on the exact name/path/task preflight before confirming.

**Voice:** "This is a native desktop tool over the installed Codex app-server. Each lot is one of my repositories, with its own task and the Codex skills I equip at prep time. Nothing here is a canned animation — and nothing starts until I confirm an exact preflight: name, path, and task for every builder."

*Recording note: the preflight and terminal deliberately show my own `~/Projects/...` paths — that's the point of a real demo. Keep everything else (browser, notifications, other windows) off screen.*

## 0:48–1:15 — Scaffolds: my repo is untouchable

**Visual:** While builders edit and chimneys smoke, open a terminal beside the app and run `git status` in one of the real repositories — clean. Show planks/blueprints as builders move.

**Voice:** "Here's the part that matters: every builder works in a scaffold — a git worktree on its own branch. My checkout stays byte-identical the whole time agents work on code I actually ship. Nothing lands without me."

## 1:15–1:50 — Inspect and install: done means merged

**Visual:** Pennants rise as builders finish. Select one — Site Inspection shows the fact line (files, ±, tests passed, duration), the builder's account, then the full diff. Click **Install in repository**. Show `git log -1` in the terminal: one squash commit in my real project. Show the workshop leveling and a plaque appearing.

**Voice:** "Completion isn't confetti — it's a pennant that means 'inspect me'. The fact line comes from telemetry the model can't fake, and the builder's account may only name files the diffstat proves it changed. One click installs a single squash commit into my real repository. Levels are earned by landed work, and the town accretes a plaque for every session that shipped."

## 1:50–2:15 — Command center flow

**Visual:** Scroll the **Village feed** — rows named with my repositories ("spotify-history · Editing vault_export.py", "Running pytest", "Tests passed"). Open **Redirect builder**, send a direction mid-turn, and show **"New direction — \<your words\>"** appear in the feed. Add a **second workshop** on one repository — two smaller houses gather on one shared parcel, "2 builders on site". Answer a builder's question in the reply card (note the context paragraph above the question); show a queued work order starting with **Start next order**. Quit and reopen — everything restores.

**Voice:** "The Village feed is one stream across every builder — real files, real commands, real test results, named per repository. Need to course-correct? Redirect a builder mid-turn without stopping it. Put two builders on one repository and they share a single parcel — two people, one job site. Questions arrive with the context to answer them; work orders queue the day's tasks and auto-start as improvements land. Quit and relaunch — the town, its levels, ledger, and queues all persist."

## 2:15–2:45 — Build story and impact

**Visual:** Briefly show README architecture and the gate commands, then end wide on the village in Wall mode.

**Voice:** "Codex with GPT-5.6 built Codeville itself — architecture, protocol bindings, the scaffold engine, the privacy registers, and the release gate that starts five real threads, proves the checkouts stay clean, installs all five improvements, and relaunches to prove persistence. Judges can reproduce all of this in one click — Codeville builds a five-repository demo village from a bundled fixture. But what you just watched was my real work landing in my real repositories. Five agents. One town. Done means merged."

## Recording checklist

- Under 3:00; judges need not watch beyond three minutes.
- On YouTube as public or unlisted (Devpost: unlisted OK), with clear audio/voiceover.
- Explicitly say **Codex** and **GPT-5.6**, and show how both were used.
- Show five live **real** projects, the clean `git status` during work, an in-app diff → Install landing a real commit, and relaunch persistence — not only code or slides.
- Keep the on-camera tasks small (docs, docstrings, a focused test) so turns finish inside the recording window; the 07-19 dogfood tasks are the proven template.
- Only shown paths are my own `~/Projects/...` — no other private data, notifications, third-party marks, copyrighted music, or unlicensed media.
- Fallback if a live run stalls on camera: **Create demo village** is the retake-safe path — same product, disposable repos. Prefer a retake on real repos first.
- Verify 1080p, audio, captions, duration, and logged-out playback.
